크롤러 아키텍쳐나 기법들은 기업들의 중요 기술로 취급되어 잘 공개 되지 않아 왔습니다. 그래서 2000년대 초반까지만 해도 '크롤러'라는 것을 어떻게 정의 내릴 것이며 어떻게 만드는지 잘 정리된 논문이 많지 않았습니다. 당시에는 Google 크롤러와 Mercator 크롤러, 이 두가지 정도가 가장 널리 알려지고 공개된 크롤러였었죠.

Mercator 크롤러가 이 글에서 소개한 Marc Najork가 만든 것입니다.

크롤러를 전문적으로 다루고 이야기 나누는 커뮤니티는 잘 본 적이 없습니다. 하지만 20여년이 흐른 지금은 오픈 소스로 진행중인 크롤러 프레임워크가 많이 공개 되어 있습니다. JAVA로 만든 Heritrix3라는 것도 있고요. Python으로 만든 Scrapy도 유명하고요. 헌데 이런 오픈 소스 프로젝트들은 꽤 오랜 시간이 지나 처음 크롤러를 공부하는 사람이 접근해서 그 구조를 모두 파악하고 즐기기엔 쉽지가 않습니다. 워낙 온갖 기능들이 추가 되어 있거든요. 심지어 Extensible, Scalable하게 만들었기 때문에 이것들을 이해하려면 Parallel Crawling에 대한 이해도 필요합니다.

go 언어에 익숙하시다면 gocolly라는 비교적 진입장벽이 낮은 크롤링 프레임 워크를 추천하고 싶습니다. 또 처음부터 크롤러라는 것이 무엇이고 어떻게 동작하면 좋은 크롤러인지 이해가 필요하다면 글에서 소개한 논문들을 읽어 보는 것도 좋습니다. 관련해서 아래 링크를 추천하고 싶습니다.

https://www.hpl.hp.com/techreports/Compaq-DEC/SRC-RR-173.pdf

Mercator 크롤러를 예로 설명하는 문서인데 진짜 Mercator라는 크롤러의 구조와 내용에 대해서 면밀히 이해하기 보다는 초반의 Introduction 2~3페이지만 잘 읽어도 크롤러에 대한 기초적인 이해가 아주 높아진다고 생각합니다. 대부분이 영문으로 된 자료지만 안타깝게도 크롤러 관련 논문을 한국어로 잘 번역한 문서는 아직 본 적이 없습니다. 설령 있다고 해도 인터넷에 공개된 것은 더더욱 없네요. 그러나 글에서 소개한 것들과 이 문서들은 정말 크롤러에 대해 좋은 내용을 담고 있기 때문에 번역기를 돌려서 읽는 한이 있더라도 읽을 가치가 있다고 생각합니다.

예상보다 많은 분들이 제 글에 호응 해 주셨기에 저 역시 이런것을 한 번 더 정리해서 또 글을 써 볼까 고민 하고 있습니다.
